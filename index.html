Tier2/SWMS Process

Complete workflow for handling SWMS requests from Salesforce Tier2 report through to CES approval.

i
Process Overview: SWMS requests come through Salesforce as automated cases. Tier2 agents process these by reviewing in SWMS Service Desk, making decisions, and coordinating goodwill compensation with CES.

Step 1: Check Tier2 Salesforce Report

○ Finding Incoming SWMS Requests

Open the Tier2 Salesforce Report to view incoming SWMS requests
Look for Automated Cases created from Salesforce when new SWMS requests arrive in the service desk
Note: Normal CEC agents typically see these cases first and tag them as "Tier2" with status set to "Tier2"
These tagged cases appear in your Tier2 report and are ready for processing

Step 2: Take Case Into Progress

○ Starting Work on a Case

Select the case from the Tier2 report
Take the case into progress
Note the SWMS number from the case details

Step 3: Process in SWMS Service Desk

○ Review and Decision in SWMS

Open SWMS Service Desk and search for the case using the SWMS number
Review all claim details, attachments, and documentation ↓ start the goodwill claim review checklist
Make your decision: Accept, Reject, or Return
Document your decision with appropriate notes

Goodwill Claim Review Checklist

○ Verify Basic Information

Dealer country (Einzelhändler Land)
First sales country matches
FIN (Vehicle Identification Number) is valid
Vehicle model and year correct
Warranty dates (Start/End) verified (Warranty NW Claims)
Mileage (Kilometerstand) within limits (Warranty NW Claims)

○ Check Service Type Classification

Serviceart = "Kunden-CX Goodwill"
Grund (Reason) is appropriate: Fahrzeugdefekt, Lieferverzug, RSA-Verzögerung, or Auftragserteilung

○ Review Labor Costs (Arbeitskosten)

Work number matches approved operations
Labor hours are reasonable
Hourly rate matches smart standards
Percentage split correct (Händler % + Hersteller % + Kunde % = 100%)

○ Review Parts Costs (Teilekosten)

Part numbers are valid smart parts
Quantities match work performed
Unit prices match smart pricing
Part category correct

○ Verify Documentation

Attachments present in "Anhang" section
CMR (shipping documents) - if applicable
Damage photos included
Damage report present
Authorization email from smart
No personal data in attachments (license plates, identifiable persons)

○ Review Notes

Dealer remarks (Anmerkungen Händler) reviewed
Manufacturer notes (Anmerkungen des Herstellers) added if needed

Step 4: Upload SWMS Report to Salesforce Case

○ Documentation Upload

In SWMS Service Desk, click Print (Drucken) to generate a PDF of the case showing your decision
Return to the Salesforce case
Upload the SWMS decision PDF as a post/attachment to the case

Step 5: Handle Based on Decision Type

+
If ACCEPTED:
  • Create a goodwill compensation request OR ask CEC to create one
  • The SWMS case itself can be used directly as the goodwill request
  • See the latest Goodwill Process KMS in Salesforce for detailed compensation steps
x
If REJECTED:
  • Add a post to the Salesforce case explaining the rejection reason
  • Close/resolve the case
←
If RETURNED:
  • Add a post to the Salesforce case explaining why it was returned
  • Close/resolve the case

Step 6: CES Notification (For Accepted Cases)

○ Notify CES After Compensation Request

Once the compensation request has been sent, wait for CES approval
After CES approval is received, tag Daniel Menner (current main CES contact as of 13.1.26) in the case
Tag as "FYI"
Attach the SWMS Decision PDF to the post
The compensation and SWMS case contain all information CES needs
i
Key Contact: Daniel Menner is the current main CES contact for FYI notifications (as of January 2026).

Process Summary Flow

1. Check Tier2 SF Report → Find tagged SWMS cases 2. Take case into progress 3. Search SWMS number in SWMS Service Desk 4. Review and make decision (Accept/Reject/Return) 5. Print SWMS case as PDF 6. Upload PDF to SF case as post 7. If Accepted: → Create goodwill/compensation request → Wait for CES approval → Tag Daniel Menner as FYI with SWMS PDF attached 8. If Rejected/Returned: → Post explanation to case → Close/resolve case

Customer Experience Goodwill Process

Complete guide for creating and managing goodwill compensation requests, including CES approval workflow.

i
What is Customer Experience Goodwill? A budget within the Customer Engagement Solutions (CES) team at smart Europe GmbH to compensate customers who had unsatisfying experiences with smart services/products. This helps maintain customer loyalty and positive brand perception.

Who Is Involved?

Party Role
Customersmart Europe GmbH customers who experienced service below standard
CECCentral point of contact - picks up complaints, requests CES approval, executes compensation
ASPAuthorized Service Partners - forward eligible cases via SWMS, support execution
CESCustomer Engagement Solutions - process/budget owner, provides approvals/rejections
MarketsCan initiate requests but must send to CEC - not authorized for final decisions

Covered Use Cases

○ Eligible for Goodwill Compensation

Delivery delay > 6 weeks caused by smart
Issues during order process blocking purchasing > 2 weeks
Defects on delivery
Incorrect deliveries (wrong colour, wrong line, etc.)
Waiting for accessories (Winter wheels, Trailer Hook, etc.) ordered via eCommerce
Vehicle issues/defects without solution at workshop > 3 weeks
Delay with Roadside Assistance > 3 hours
More than 3 workshop visits for same defects within 4 months
Special cases at instigation of Area Sales Manager or market CEO
Certain mobility cases (not covered by RSA/workshop mobility)
RSA related costs exceeding limits
x
NOT Covered:
  • Mistakes by customer or dealer
  • Customers who already received a benefit from smart
  • Minor inconveniences solved within 3 weeks
  • Non-customer cases (market vehicles, demo cars)
  • Defect not yet fixed (wait until root cause is resolved)
  • Missed promotions by a few days
  • Frozen doors

Compensation Options

Type Options Notes
Vouchers
(Primary)
  • Merchandise Vouchers (priority)
  • DCS Charging Vouchers (fallback)
 Values: 50-500 EUR. Managed by CES, sent via email by CEC
Discounts
  • Accessories from smart store
  • Workshop visit costs
 Must go through SWMS, needs smart Europe approval
Other
(Rare)
  • Payout (requires market FC approval)
  • Mobility solutions
 Cash refunds should be last option

Compensation Decision Matrix

Calculate points based on severity, customer type, impact, and repetitiveness to determine appropriate compensation:

Points Score < 5 5-8 9-11 12
Voucher 50 EUR 100 EUR 150-200 EUR 250-500 EUR
Discount (accessories) - up to 100 EUR up to 200 EUR up to 500 EUR
Discount (workshop) - - up to 200 EUR up to 500 EUR

Points Calculation:

Category 1 Point 2 Points 3 Points
SeverityMinor inconvenienceSignificant disruptionSafety/critical concern
Customer TypeLead customerB2C customerB2B customer

CES Approval Process

○ Step-by-Step Approval Workflow

1. Document case in Salesforce (all goodwill cases must be documented)
2. Check compensation matrix to determine appropriate offer
3. Tag your CES Ops Specialist to discuss options before suggesting to customer
4. Create Goodwill approval request in Salesforce with:
  • Applicable use case (copy/paste from covered cases)
  • Proof points explaining why use case applies
  • Request type and justification
5. Submit for approval - CES Specialist reviews and approves/rejects
6. If >= 500 EUR: Send Goodwill Agreement to customer for signature
7. Execute compensation (send voucher code / inform retailer for discount)

CES Ops Specialists by Market

Specialist Markets
FaizaUK
Daniel M.DACH (DE, AT, CH), NL
Daniel B.ES, PT, IT
CarlaFR, BE, SE
!
Important Rules:
  • Prioritize Merchandise vouchers as primary compensation
  • Use DCS vouchers as fallback if merchandise unsuitable
  • Any non-merchandise compensation must be raised to CES first
  • Don't promise compensation before CES approval
  • Claims in SWMS are valid for 90 days only

Goodwill Agreement Requirement

A signed Goodwill Agreement is required when:

  • Compensation value is >= 500 EUR/CHF/GBP
  • Compensation item is an accessory not offered in official smart store (e.g., floor mats, wallbox)

Process: Customer signs first, then market signs. Download templates from KMS (search "Goodwill Clause").

Mark post as "FYI" - no need for detailed information

Cost Review & FRT Management

Labor costs, Flat Rate Time (FRT), Work Units (AW), and cost distribution.

Labor costs in SWMS are calculated based on either FRT (Flat Rate Time) for time-based billing or AW (Arbeitswerte/Work Units) for unit-based billing.

Labor Cost Fields

Field German Description
Causal WorkKausale ArbeitRoot cause work item
Work NumberArbeitsnummerOperation code
DescriptionBeschreibung VorgangWork description
Work HoursArbeitsstundenHours claimed
Unit PriceArbeit EinzelpreisHourly rate

Cost Distribution

Each cost line must have percentage distribution totaling 100%:

Party Field Typical Range
DealerHändlerprozentsatz0-50%
ManufacturerHerstellerprozentsatz50-100%
CustomerKundenprozentsatz0-50%

Claimed vs. Approved Amounts

The system tracks two sets of values. Always compare:

  • Arbeitsstunden vs. Genehmigte Arbeitsstunden
  • Stückpreis vs. Genehmigter Stückpreis
  • Mg. vs. Genehmigte Mg.

○ Cost Verification Checklist

Work number exists in smart system
Hours are reasonable for the operation
Hourly rate matches current standards
Percentage split totals 100%
Approved amounts don't exceed claimed
Work description matches claim type

Status Codes

Understanding claim statuses and their meanings.

Status German Description
Processing Wird verarbeitet Claim is being reviewed - initial state
Under Review Under granskning Awaiting decision - being evaluated
Released Freigegeben Approved for payment
Rejected Abgelehnt Claim denied

Status Flow

1. Wird verarbeitet (Processing) → Initial state 2. Under granskning (Under Review) → Being evaluated 3. Freigegeben (Released) → Approved OR Abgelehnt (Rejected) → Denied
